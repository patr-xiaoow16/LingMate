# 产品设计文档 - 听伴 LingMate

**AI Co-pilot for Immersive Listening（AI 沉浸式精听伴侣）**

Ling 很像 language / linguistic 里的前缀联想，容易让人想到“语言、英语、听力”这类学习场景。它不是一个冷冰冰的题库，也不是纯播放器，更像一个一直陪你做精听的人，帮我反复听、拆句、跟进、纠错，让精听这件本来有点枯燥的事情，变得更有陪伴感。

## 核心痛点

### 痛点1：“听了等于没听”——被动收听，缺乏精听方法论
大多数用户的“精听”实际上只是“泛听”——打开音频就听，没有明确的学习步骤和目标。听了 30 分钟发现什么都没记住，因为大脑在“自动导航”模式下不会主动加工语言输入。真正的精听需要一套结构化的方法，但用户自己很难执行这么复杂的流程。

### 痛点2：“词都认识，连起来就听不懂”——语音现象解码缺失
很多用户词汇量其实不小，但听力依然很弱。根本原因是连读（gonna = going to）、弱读（him 变成 'im）、T 音变化（water → wader）、吞音等语音现象没有被系统学习过。传统工具无法帮用户识别和练习这些现象。

### 痛点3：“会听不会用”——从输入到输出的鸿沟
用户听懂了 “I’m feeling a bit under the weather”，但自己说的时候还是会说 “I have a little sick”。听懂 ≠ 学会，用户缺乏从“输入理解”到“输出运用”的桥梁。传统精听工具只做了“听”这一步，没有引导用户做 Chinglish 对照、场景迁移、句型内化和输出练习。

### 痛点4：“听懂字面意思，听不懂言外之意”——语用理解缺失
英语中大量的意思藏在语气、语调和潜台词中。比如请假时说 “I’m not feeling great” 与 “I’m really not well at all” 传递的信息强度完全不同。用户缺乏对这种“语用层”的感知训练，导致在真实场景中无法理解对方的真实意图。

### 痛点5：“学完就忘”——缺乏系统化的复习与固化机制
即使用户在某次精听中学到了新表达，也会因为没有系统的复习机制而快速遗忘。词汇本里存了几百个词但从来不复习，新学的句型没有被融入到下一次的学习中。学习闭环的“最后一公里”始终缺失。

## 机会点

| 痛点 | 机会点 | AI 如何实现 |
|---|---|---|
| 痛点① | AI 作为“精听教练”，自动编排八步学习流程 | 用户只需导入材料，AI 自动将其拆解为结构化的 8 个学习模块，无需用户自己设计学习流程 |
| 痛点② | AI 自动标注连读/弱读/音变，生成对比练习 | 自动识别语音现象并可视化标注，生成针对性的跟读练习 |
| 痛点③ | AI 生成 Chinglish 对照表和场景模板 | 自动对比中式英文 vs 地道表达，并基于场景生成可复用的口语模板 |
| 痛点④ | AI 解读语气/潜台词，训练语用感知 | 分析说话人的语气、意图和情绪，让用户理解“话背后的意思” |
| 痛点⑤ | AI 建立个人词汇本，智能安排复习节奏 | 跨课程记忆薄弱点，按艾宾浩斯曲线智能复现，并在新材料中融入旧词 |

## 产品核心价值

帮助英语学习者，在收听任意原版英文材料时，通过八步沉浸式精听法，将每一段音频转化为一堂覆盖“从听懂到学会再到会用”的全链路个性化精听课。

## 核心功能与流程设计

## 八步沉浸式精听法

沉浸听力 → 核心词汇 → Chinglish 对照 → 场景迁移 → 听力解码 → 潜台词与语气 → 句型内化 → 输出练习

## 用户行为路径

| 步骤 | 用户行为 | 系统响应 |
|---|---|---|
| 导入材料 | 粘贴链接 / 从推荐库选择 / 上传音频 | 自动转写、难度分级、生成八步学习方案 |
| Module 1: 沉浸听力 | 纯听 2-3 遍，感受语速和情绪 | 播放音频，提供遍数记录和进度条 |
| Module 2: 核心词汇深度拆解 | 查看 AI 提取的关键表达，学习释义和用法 | 提取高频/关键表达，生成释义、场景和例句 |
| Module 3: Chinglish 对照 | 对比中式英文 vs 地道说法 | 生成“我会说” vs “原文说”的对照表 |
| Module 4: 场景映射 | 获取可直接复用的场景句型模板 | 识别场景，生成 3-4 个可复用的职场/生活口语模板 |
| Module 5: 听力解码 | 逐句跟读，理解连读/弱读/音变 | 标注语音现象，提供单句循环播放 |
| Module 6: 潜台词与语气 | 理解说话人的真实意图和情绪 | 分析语气、意图和潜台词，配合中文解读 |
| Module 7: 地道句型拆解 | 学习可复用的句型“骨架” | 抽取句型结构，生成“骨架 + 换词”的练习 |
| Module 8: 输出练习 | 写/说英文输出，应用所学内容 | 出题、评分、给反馈 |
| 学习报告 + 复习 | 查看报告，完成 AI 生成的复习任务 | 生成学习报告 + 艾宾浩斯复习卡片 |

## 功能模块详细设计

## 模块 A：智能内容推荐与预处理

### 材料导入方式（三种入口）
- **链接粘贴**：支持 YouTube / Spotify Podcast / Apple Podcast / TED / BBC 等主流平台链接，系统自动抽取音频并调用 Whisper API 进行高精度转写
- **智能推荐库**：系统按场景（商务/日常/学术/娱乐）和难度等级分类的预置内容库，基于用户历史表现和兴趣标签推荐材料
- **本地上传**：支持 MP3 / MP4 / M4A 文件直接上传

### AI 预处理流水线（用户导入后 30 秒内完成）
1. **语音转写**：Whisper API 生成带时间戳的逐句转写结果
2. **难度分级**：LLM 综合分析词汇难度（CEFR A1-C2）、语速（WPM）、句式复杂度，评定综合难度等级并判断是否匹配用户水平
3. **关键表达提取**：LLM 提取核心词汇、俗语、固定搭配，并标注 CEFR 等级、中文释义、使用场景
4. **场景识别**：LLM 判断材料场景（请假、面试、投诉、日常对话等），为后续场景映射和句型拆解做准备
5. **语音现象标注**：LLM 识别文本中的连读、弱读、T 音变化、吞音等位置
6. **八步学习方案生成**：基于以上分析结果，AI 自动为这段材料编排完整的八步学习内容

## 模块 B：沉浸式交互精听——八步沉浸式精听法

### Module 1：沉浸听力（Immersive Listening）
**设计理念**：先不看文字，纯用耳朵感受语速、语气和整体意思。
- 音频播放器，无字幕，仅显示进度条和遍数计数器
- 支持 0.75x / 1.0x / 1.25x 变速播放
- 每遍听完后，AI 弹出 1-2 个简单的“大意理解检查”问题（如“这段音频主要在讨论什么？”）
- 建议听 2-3 遍后进入下一模块，但用户可随时跳过

### Module 2：核心词汇深度拆解（Vocabulary Deep Dive）
**设计理念**：不是简单列生词表，而是对关键表达进行深度拆解。
- AI 自动提取材料中最关键的 5-8 个表达（优先选择高频、核心、用户未掌握的）
- 每个表达显示为一张“词汇卡片”，包含：中文释义、原音频位置（可点击回听）、使用场景说明、AI 生成的 2 个新例句
- 交互：用户可以尝试自己造句，AI 实时纠正并给出改进建议
- 用户可标记“已掌握”或“需复习”，标记结果自动同步到个人词汇本

### Module 3：Chinglish 对照（Native vs Chinglish）
**设计理念**：直观展示“我会怎么说” vs “母语者怎么说”的差异。
- AI 基于材料中的地道表达，自动生成 4-6 组“中式英文 vs 地道英文”对照
- 左右分栏显示：左侧 ✖️ 中式表达（红色），右侧 ✔️ 地道表达（绿色）
- 每组对照附带 AI 解说：为什么这样说更自然，背后的思维差异是什么
- 交互：用户可输入自己的表达，AI 判断是否 Chinglish 并给出地道替代

### Module 4：场景映射（Scene Mapping）
**设计理念**：将学到的表达落地到用户的真实生活/工作场景中，“学了就能用”。
- AI 识别材料场景（如：请假、商务谈判、日常对话、学术讨论等）
- 生成 3-4 个“可以直接搬去用”的场景模板句，每句附带“何时用 / 说给谁听 / 语气建议”
- 用户可按自己的场景偏好（职场/日常/学术）筛选显示
- 交互：用户可描述自己的场景（如“我想向老板请病假”），AI 基于本课所学生成定制化的口语脚本

### Module 5：听力解码（Listening Decoder）
**设计理念**：解决“词都认识但听不懂”的根本问题——系统学习语音现象。
- AI 自动标注材料中的连读（gonna、wanna）、弱读（him → 'im）、T 音变化（water → wader）、吞音等
- 显示字幕 + 语音标注双层视图：上层显示原文，下层显示“实际听起来像什么”
- 支持单句循环播放 + 变速，方便逐句跟读
- 交互：用户跟读后，AI 可评估跟读准确度（V2.0 语音识别功能）

### Module 6：潜台词与语气（Subtext & Tone）
**设计理念**：超越字面意思，理解说话人“真正想表达的”。
- AI 选取材料中 3-5 个“有潜台词”的关键句子
- 每个句子展示：原文 + 语气标签（真诚、委婉、试探、不满等） + AI 解读“说话人到底在暗示什么”
- 对比展示：同一句话用不同语气说，传递的信息有何不同
- 交互：AI 出题——“你觉得说话人这里是什么意思？”，用户选择/回答后 AI 解答并解释

### Module 7：地道句型拆解（Pattern Extraction）
**设计理念**：将实用句型抽象为骨架，用户学会骨架就能套用到无数场景。
- AI 从材料中抽取 3-5 个高价值句型，拆解为“骨架 + 可替换部分”
- 示例：原句 “I’m calling in sick today” → 骨架 “I’m [doing X] today” → 换词练习：“I’m working from home today” / “I’m taking the day off today”
- 交互：用户用骨架造句，AI 实时判断是否自然并给出改进建议
- 难度递进：简单替换 → 变化句式 → 融合多个句型的复杂表达

### Module 8：输出练习（Output Challenge）
**设计理念**：学习的最终检验是用出来。
- AI 基于本次学习内容生成 2-3 个输出任务（难度递增）：
  - 填空复述：给出学过的句型骨架，用户填入关键词
  - 场景写作：“假设你要向老板请假，请用今天学的表达写一封邮件”
  - 口语模拟：AI 扮演对话另一方，用户用文字或语音进行角色扮演（V2.0）
- AI 对每个输出任务给出详细反馈：词汇使用是否准确、语法是否正确、表达是否地道、改进建议

## 模块 C：个性化生成复习模块

### 学习报告（每次精听后自动生成）
- 本次学习时长、完成模块数、交互次数
- 听懂率评估（基于 Module 1 问答、Module 6 潜台词理解、Module 8 输出质量综合评定）
- 薄弱点雷达图：词汇、连读弱读、句式理解、语用感知、输出能力五个维度
- 新掌握表达列表（自动同步到个人词汇本）
- 与上次精听的对比进步

### AI 智能复习系统
- **艾宾浩斯词汇卡片**：新学表达自动加入复习队列，按遗忘曲线规律在第 1/3/7/14/30 天推送复习
- **场景化复习**：复习时不是简单英→中翻译，而是 AI 生成新的场景对话来测试用户是否真正掌握
- **跨课程融合**：新一次精听时，如果材料中出现了用户词汇本中的旧词，AI 会主动提醒“这个表达你上次在 XX 材料中学过，还记得吗？”
- **周报/月报**：汇总用户的学习进度、词汇增长、薄弱点变化趋势

## 用户增长

### 策略一：“精听笔记”社交裂变
- **用户触发**：完成精听后，系统自动生成一张精美的「精听笔记卡」（包含：材料信息 + 3 个地道表达 + 1 个 Chinglish 对照 + 学习统计）
- **裂变钩子**：分享到朋友圈/小红书，好友扫码可免费解锁同一段材料的精听课，分享者获得额外 AI 时长奖励
- **增长飞轮**：用户学习 → 生成笔记卡 → 分享获取奖励 → 新用户体验 → 转化注册 → 新用户学习
- **预期效果**：每月新增用户中 15-20% 来自社交裂变，获客成本趋近于零

### 策略二：“精听方法论”内容营销
- **短视频矩阵**：在抖音/小红书/B站发布“用八步法听懂 XX”系列短视频。每期选一段热门英文材料（美剧片段、TED 片段、热点新闻），展示产品如何拆解，形成“看完想试”的冲动
- **公众号/知乎深度内容**：发布“为什么听了很多年英语听力还是很弱”“连读弱读完全指南”等深度文章，在文末自然导流到产品
- **预期效果**：建立“精听 = 英语流利说”的品类心智，自然搜索流量提升 30%+

### 策略三：“播客精听”跨界合作
- **合作形式**：与商务英语播客、英语学习类播客等达成合作，在其节目页面嵌入“用精听伴侣学习这一期”按钮
- **双赢逻辑**：播客获得流量和用户粘性提升，我们获得精准用户和优质内容
- **预期效果**：每个合作播客带来 500-2000 注册用户，且转化率高于广告渠道

### 策略四：付费广告精准投放
- **广告形式**：15 秒屏录视频广告，直接展示产品内的八步流程
- **投放渠道**：抖音信息流（定向英语学习兴趣标签）、小红书信息流（定向留学/英语学习标签）、微信朋友圈广告
- **ROI 控制**：先小规模测试多组素材，找到 CPA < 15 元的素材组合后再规模化投放

## 商业化模式

### 方案 A：“Freemium + AI 时长包”增值服务模式
基础功能免费（每天 1 次完整精听，含全部 8 个模块），高级功能付费解锁。

**免费版包含：**
- 每天 1 次完整八步精听（含预处理 + 全部 8 个模块）
- 基础学习报告（单次报告，不含历史趋势）
- 分享精听笔记卡片功能

**付费版解锁（两种计费）：**
- **AI 时长包**：按次购买，10 次包 / 30 次包 / 100 次包，单次价格随包大小递减
- **月卡/年卡**：无限次数精听 + 个人词汇本 + 艾宾浩斯复习 + 进度报告 + 专属推荐库

### 方案 B：“流利说 Pro+”订阅等级模式
将精听伴侣作为新的“流利说 Pro+”会员套餐的核心卖点，定价高于现有 Pro 会员。

### 方案 C：“免费钩子”引流转化模式
精听伴侣全功能免费，作为吸引新用户的“超级钩子”，通过它拉升英语流利说整体的注册量和付费会员转化率。

### 方案 D：B2B 企业/院校解决方案
将八步精听法的 AI 能力包装为 SaaS 产品，向企业培训部门、英语培训机构、高校外语学院销售。

## 推荐方案

### 方案 A：“Freemium + AI 时长包”

**理由一：最符合“新功能冷启动”的增长需求**
作为一个全新的 Mini-App，最大的挑战不是变现，而是“让足够多的用户体验到产品价值”。方案 A 的免费版提供完整的八步精听体验（每天 1 次），这意味着：
- 用户可以在第一次使用中就体验到完整的“啊哈时刻”
- 体验足够好才会产生“明天还想用”的欲望，而“每天只能用 1 次”形成的稀缺感正好驱动付费转化
- 相比方案 B（高付费墙）和方案 C（全免费无收入），这是“用户增长”和“收入”的最优平衡点

**理由二：“时长包 + 订阅”双轨制覆盖两类用户**
精听学习有两种典型用户模式：
- **低频学习者（如老王）**：每周只学 2-3 次，他们不想买月卡，“AI 时长包”允许他们按需付费，心理负担低
- **高频学习者（如 Mia）**：每天都学，月卡/年卡明显更划算，并且订阅提供的“个人词汇本 + 艾宾浩斯复习”对他们很有价值

这种双轨制避免了“一刀切”定价对不同用户的不适配。

**理由三：AI 成本可控，商业模型可持续**
相比方案 C（全免费），方案 A 通过“每天 1 次免费”设定了清晰的 AI 调用成本上限。

**理由四：为 B2B 开辟第二增长曲线留空间**
当 C 端产品经过 V1.0-V1.5 的打磨后，八步精听法将形成一套成熟的教学方法论和技术能力，此时引入方案 D（B2B）非常自然：
- **企业培训**：为外企提供“员工英语听力提升方案”，基于其行业材料（如金融报告、技术会议录音）定制精听课程
- **院校合作**：为高校英语学院提供“智能听力教学工具”，老师可指定材料，AI 自动生成八步精听作业

这样形成“C 端 Freemium 拉新 + B 端高客单价变现”的双引擎结构，商业模型更稳健。
